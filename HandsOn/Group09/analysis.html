1. Dataset analysis

1.1. Entities

The dataset contains the following columns:

  • X: Longitude component expressed as a double.
  • Y: Latitude component expressed as a double.
  • OBJECTID: Identifier given to each row of the document, starting by 1 and increasing one unit per subsecuent row.
  • IDPTOVENTA:Identifier for each selling point. It is composed by the prefix "FB_" and a six digits hexadecimal number.
  • Municipio: Municipality where the selling point is located.
  • Provincia: Province of the selling point.
  • Código_Postal: Postal code of the selling point.
  • Dirección: Name of the street where the selling point is located.
  • Latitud: Latitude component of the selling point. It is the same value as the Y component but with some decimals rounded.
  • Longitud: Longitude component of the selling point. It is the same value as the Y component but with some decimals rounded.

2. License analysis

2.1. Publisher:

The publisher of this dataset is the Consorcio Regional de Transportes de Madrid (CRTM) who published the data in its opendata portal.

2.2. Rightsholder:

The owner of the data is the same as the publisher, the Consorcio Regional de Transportes de Madrid (CRTM).

2.3. License:

The dataset comes with a LDA license (Licencia de Datos Abiertos) which allows the following (among others):

  • Download the data for free.
  • Share the data under the same license.
The license also has the following restrictions (among others):
  • It is not allowed to alter the data in order to modify the represented knowledge.
  • The CRTM must be cited as the source of the data.
  • Any application that utilizes this data must include the text "Powered by CRTM" and a link to http://www.crtm.es/.
  • It is not allowed to alter/delete the metadata regarding the updating date and license.
More information about the license can be found here.

3. Naming estrategy

The base URI is http://pt-recharge-points.org. The following URIs represent all the resouces. Note that "Store-ID" represents the value of the column "IDPTOVENTA" of the dataset.

  • Cigar store: http://pt-recharge-points.org/resource/cigar-stores/{Store-ID}
  • Address: http://pt-recharge-points.org/resource/cigar-stores/{Store-ID}/address
  • Latitude: http://pt-recharge-points.org/resource/cigar-stores/{Store-ID}/location/latitude
  • Longitude: http://pt-recharge-points.org/resource/cigar-stores/{Store-ID}/location/longitude
  • Postal code: http://pt-recharge-points.org/resource/cigar-stores/{Store-ID}/location/postal-code
  • Municipality: http://pt-recharge-points.org/resource/cigar-stores/{Store-ID}/location/municipality
Since all instances correspond to cigar stores located in the province of Madrid, we will ignore the "Province" field.
